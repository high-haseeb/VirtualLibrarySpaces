# Virtual Libray Space

### Brief

Users start in a digital start room and then select one of the four libraries. Each library has its
own virtual room, visually similar in structure but with different colors/branding and some unique 
elements.

The rooms are not 1:1 copies of real buildings, but specially designed digital spaces (roughly the 
size of a small shop) with clear, simple navigation using mouse/keyboard.

Inside each room, the library’s digital services are placed as interactive elements, for example:

* A digital bookshelf that links to the online catalog and loan functions
* Areas for digital media (e‑books, magazines)
* Posters/walls showing events, with direct online booking/participation
* Access to the user’s account (loans, renewals, personal data, etc.)
* Community features like a digital notice board, book clubs, exchange formats

The system can be connected to the existing library system WinBIAP (by Datatronics) to use:

* Online catalog & digital lending
* Access to digital media
* User accounts and borrower cards
* Event offers incl. booking & participation

Additionally, a digital assistant (chat) should be integrated in the virtual rooms, optionally as an avatar (e.g. a librarian figure) with a text‑based chat interface. This can be based on AI/LLM technology; the exact technical solution is open.

The whole solution should be built on an open‑source technology stack.

* Frontend implementation of the virtual rooms in the browser (e.g. JavaScript/TypeScript, Web‑3D with Three.js/Babylon.js or comparable, plus a modern web framework like React/Vue/Svelte).
* Integration of the library services via APIs (WinBIAP and possibly other web services).
* Implementation or integration of a text‑based chat assistant (LLM‑based), including the web UI and the connection to a backend/API.
